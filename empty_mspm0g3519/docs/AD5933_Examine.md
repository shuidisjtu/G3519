# AD5933 功能验证指南

> 编写日期：2026-07-22
> 验证工具：AD2（Analog Discovery 2）+ WaveForms
> 无需万用表——I2C ACK 即可确认硬件连通性。

---

## 一、验证前准备

### 1.1 固件

编译烧录当前项目代码。开机动画后进入主菜单，选择 **AD5933 Test**。

> `action_ad5933_test()` 已改为调用库函数（`tsp_ad5933_init` / `read_temperature` / `set_sweep` / `start_sweep`），验证 P2/P3/P6 修复的同时在 LCD 上显示实时 Real/Imag/Magnitude。

### 1.2 AD2 接线

| AD2 端口 | 主板连接点 | 用途 |
|----------|-----------|------|
| D0 | PA29 或 J19-4 | I2C SCL |
| D1 | PA30 或 J19-6 | I2C SDA |
| 1+ | J15-5 (VOUT_BUF) | 激励波形 |
| 1- | J15-2 (GND) | |
| 2+ | J15-4 (SENSE_IN) | 回采波形（第五步） |
| 2- | J15 GND | |
| GND | J15 GND | 共地参考（必须连接） |

### 1.3 J15 第五步接线（仅第五步）

```
J15-5 (VOUT_BUF) ──[10kΩ 电阻]── J15-4 (SENSE_IN)
```

> 前四步不需要在 J15-4/5 之间接被测件；第四步只需将 AD2 的 `1+`/`1-` 接到 J15-5/J15 GND。AD2 的 Scope 为**全差分输入**，即使 `1-`、`2-` 已接到板上 GND，仍应将 AD2 的 `GND` 端子与主板 GND 相连，以建立稳定的共模参考。

## 二、已通过验证项（2026-07-22）

| 步骤 | 验证项 | 工具 | 结果 |
|------|--------|------|------|
| P2 | CTRL_H 写入（16-bit 宏 `>>8`） | LCD Row 1 | `C: B0 08` ✅ |
| P6 | 温度读取（14-bit 掩码 + 符号扩展） | LCD Row 2 | `32.1°C`，手指按压后上升 ✅ |
| I2C | 总线通信 | AD2 Logic | `0x0D WR`，各字节 ACK ✅ |
| P3 | 频率字公式（MCLK/4 分频） | AD2 Scope CH1 | 1.000 kHz / ~2Vpp 正弦波 ✅ |

> 以上四项在 `action_ad5933_test()` 中一站式验证：进入菜单 → 自动 init → 显示 CTRL 回读 → 测温（每 1s 刷新）→ 启动扫频 → Real/Imag/Mag 实时刷新。详见 `AD5933_Use.md`。

## 三、待验证：已知电阻校准

| 工具 | 模式 | 接线 |
|------|------|------|
| AD2 | Scope 双通道 | J15-5 ↔ [10kΩ] ↔ J15-4 |

**目的**：验证完整模拟链路（激励 → DUT → 回采 → ADC → DFT）。

**AD2 操作**：保留第四步的 Scope 设置并启用 CH2；确认 `2+` 接 J15-4、`2-` 接 J15 GND。触发仍使用 CH1，使两路波形稳定显示；分别为 CH1、CH2 添加 **Frequency** 测量。Scope、Voltmeter、Spectrum Analyzer、Network Analyzer 共用 AD2 的模拟输入资源，因此本步骤只运行 Scope，不要同时运行这些会占用 Scope 的仪器。

#### ✅ 正确时 AD2 Scope 双通道应显示

```
┌──────────────────────────────────────┐
│ CH1 (J15-5, 激励): ~2 Vpp 正弦      │
│   /\      /\      /\      /\        │
│  /  \    /  \    /  \    /  \       │
│ /    \  /    \  /    \  /    \      │
│                                      │
│ CH2 (J15-4, 回采): 正弦，幅值 < CH1 │
│   /\      /\      /\      /\        │
│  /  \    /  \    /  \    /  \       │ ← 幅值由前端增益和回采网络决定
│ /    \  /    \  /    \  /    \      │
│                                      │
│ Freq: 1.000 kHz（两通道一致）        │
└──────────────────────────────────────┘
```

#### ✅ 正确时 LCD Row 4–6 应显示

```
Real:  04321          ← 非零，随电阻变化
Imag:  01234          ← 非零（含系统相位偏移）
Mag :  12345          ← 非零且稳定，同电阻值下抖动 <5%
```

**快速验证**：换另一个已知电阻（如 4.7kΩ）→ Magnitude 应有明显变化。

#### ❌ 对照

| 现象 | 原因 |
|------|------|
| CH2 无波形（0V 直线） | J15 接线开路 / 电阻未可靠接触 |
| CH2 波形与 CH1 完全相同 | J15-4/5 短路或电阻为 0Ω |
| Magnitude 恒为 0 | DFT 数据无效 → 检查 STATUS D1 |
| Magnitude 大幅跳动 | J15 飞线过长 → 缩短线缆，用双绞线 |

#### GainFactor 标定（记录备用）

```
GF = 1 / (R_cal × Magnitude)
Z_unknown = 1 / (GF × Magnitude_unknown)
```

## 五、故障速查

| 现象 | 最可能原因 | 排查 |
|------|-----------|------|
| 第一步无 ACK | J19 跳线错位 | 断电重插 J19 跳线帽，确认 pin 对正确 |
| 第一步 SDA 始终低 | 总线短路 | AD2 Scope 测 SDA 对 GND 电压，应为 3.3V（上拉） |
| 第二步 `C: 00 08` | P2 修复未烧录 | 重新编译，确认 `tsp_ad5933.h` 中 `STANDBY` 宏为 `0xB000` |
| 第四步 250 Hz | P3 修复未烧录 | 确认 `tsp_ad5933.c` 中为 `536870912ULL`（非 `134217728ULL`） |
| 第四步无波形 | DDS 未启动 | 确认菜单函数调用了 `set_sweep()` → `start_sweep()` |
| 第五步 Mag=0 | J15 接线开路 | Scope CH2 确认有无信号，重插电阻引脚 |
| 第五步 Mag 跳动大 | 飞线过长引入噪声 | J15 到电阻的线 <10cm，用双绞线或同轴线 |
