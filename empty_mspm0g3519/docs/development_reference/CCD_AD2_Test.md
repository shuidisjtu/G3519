# CCD 代码功能验证 — AD2 测试方案 (无传感器)

> 使用 Analog Discovery 2 模拟 CCD 模拟输出，验证驱动代码和应用层逻辑的正确性。
> 适用于 CCD 传感器模组尚未到手的阶段。

## 前置条件

- 主板已烧录最新固件，能正常进入菜单并选择 "CCD Test"
- Analog Discovery 2 + WaveForms 软件已就绪
- 主板与 AD2 共地（**必须先接 GND**）

## 接线总表

以 CCD1 (J3) 为主测通道：

| AD2 端口 | 接到 | 信号 | 说明 |
|----------|------|------|------|
| GND (黑) | J3 GND | 数字+模拟共地 | **必须首先连接** |
| DIO 0 (粉) | PC9 / J3 SI 脚 | SI 信号 | 逻辑分析仪输入 |
| DIO 1 (绿) | PB20 / J3 CLK 脚 | CLK 信号 | 逻辑分析仪输入 |
| 1+ (橙) | PB18 / J3 AO 脚 | AO 模拟观测 | 示波器输入 (可选) |
| W1 (黄) | PB18 / J3 AO 脚 | 模拟注入 | 波形发生器输出 |

> J3 引脚顺序需对照 PCB 丝印确认。信号对应关系：AO=PB18, CLK=PB20, SI=PC9, VDD=3V3, GND。

CCD2 测试时改接 J17：SI=PC4, CLK=PC5, AO=PB17，AD2 接线相应调整。

## 测试 A：SI/CLK 时序验证

**目的**：确认代码产生的数字信号序列正确。

### 接线

仅需 DIO 0 → SI, DIO 1 → CLK, GND。不需要波形发生器。

### WaveForms 设置

1. 打开 **Logic** (逻辑分析仪)
2. 通道：DIO 0 = SI, DIO 1 = CLK
3. 采样率：10 MHz (足够抓 ~3us 的 delay 脉冲)
4. 触发：DIO 0 上升沿 (SI 起始脉冲)
5. 采集深度：至少 200ms (覆盖 flush + 曝光 10ms + 读出)

### 操作步骤

1. 进入 CCD Test 菜单（默认连续模式、CCD1、曝光 10ms）
2. AD2 触发采集
3. 捕获一帧完整波形

### 检查项

| # | 检查点 | 预期 | 判定方法 |
|---|--------|------|---------|
| A1 | flush 起始：SI↑ 先于或同时于 CLK↑ | SI 在 CLK 上升沿时为高电平 | 对比两通道波形时间轴 |
| A2 | flush CLK 脉冲数 | 恰好 128 个 | 用 WaveForms 计数器测 CLK 脉冲数 |
| A3 | 曝光间隔 | SI 落后于 flush 最后 CLK↓ 约 10ms | 测量 flush 结束 → 读出 SI↑ 的时间间隔 |
| A4 | 读出起始脉冲 | 与 flush 起始脉冲模式相同 | SI↑→CLK↑→SI↓→CLK↓ |
| A5 | 读出 CLK 脉冲数 | 128 + 1 (第 129 个终止脉冲) = 129 个 | 计数从读出 SI↑ 之后的 CLK 脉冲 |
| A6 | CLK 脉宽 | 高电平 ~3us, 低电平 ~3us (来自 delay loop) | 测量单个 CLK 脉冲 |
| A7 | CLK 抖动 | 读出期间个别 CLK 周期可能偏长 (SysTick 中断) | 测量 CLK 周期标准差，确认无异常长周期 |

### 变化测试

- 修改曝光时间 (S0/S1 调节)：重复采集，确认 A3 的间隔跟随变化
- 切换 CCD2 (S2)：改接 J17 的 SI/CLK，确认信号出现在 PC4/PC5 上

## 测试 B：ADC 精度验证 (DC 注入)

**目的**：确认 ADC 读数与输入电压线性对应。

### 接线

W1 → PB18 (J3 AO)，GND 已连。DIO 可保留。

### WaveForms 设置

1. 打开 **Wavegen**
2. 波形：DC
3. 偏移：从 0V 开始

### 操作步骤 & 检查项

在 CCD Test 连续模式下，依次设置 W1 为以下 DC 电压，观察 LCD 显示：

| W1 电压 | 预期 ADC 值 (12-bit) | 预期 LCD 显示 | 波形位置 |
|---------|---------------------|--------------|---------|
| 0.00 V | ~0 | Max/Min/Avg ≈ 0, 波形贴底 (y=111) | 底部 |
| 0.80 V | ~993 (0.80/3.3×4096) | M:0993 m:0993 Avg:0993 | 约 1/4 高度 |
| 1.65 V | ~2048 | M:2048 m:2048 Avg:2048 | 正中间 |
| 2.50 V | ~3103 | M:3103 m:3103 Avg:3103 | 约 3/4 高度 |
| 3.30 V | ~4095 | M:4095 m:4095 Avg:4095 | 顶部 (y=32) |

> 注意：实际值可能有 ±20 LSB 误差（ADC 量化噪声+参考电压偏差）。

### 判定标准

- 128 个像素应全部显示**相同值**（DC 输入 → 平线波形）
- Max = Min = Avg（允许 ±5 差异为噪声）
- 波形在 LCD 上为水平直线
- 电压与 ADC 值成**线性关系**

### 故障排查

| 现象 | 可能原因 |
|------|---------|
| 所有值 = 0 | W1 未连接/GND 未连/ADC 未初始化 |
| 值固定不变，不随电压变化 | VREF 配置错误 (已修复为 VDDA) |
| 值随机跳动 | 接线不良/电磁干扰 |
| Max ≠ Min (大波动) | CLK 信号串扰到 AO 线（检查走线） |

## 测试 C：波形响应验证 (慢速三角波注入)

**目的**：确认 LCD 波形绘制和动态更新正确。

### WaveForms 设置

1. Wavegen: 三角波 (Triangle / Ramp)
2. 频率：**200 Hz**（周期 5ms ≈ 读出时间，一帧内看到完整斜坡）
3. 幅度：3.3 Vpp
4. 偏移：1.65 V（使波形在 0V~3.3V 之间）

### 预期结果

LCD 波形区域应显示一条**从底到顶（或顶到底）的斜线**，因为：
- 128 个像素在 ~5ms 内依次读出
- 三角波周期 5ms → 一帧期间电压从 0V 线性上升到 3.3V
- 像素 0 读到低值，像素 127 读到高值

### 检查项

| # | 检查点 | 预期 |
|---|--------|------|
| C1 | 波形形状 | 近似直线斜坡 (底→顶 或 顶→底) |
| C2 | Max vs Min | Max ≈ 4095, Min ≈ 0 (覆盖全量程) |
| C3 | 连续帧更新 | 斜线方向可能帧间交替 (三角波上升/下降半周) |
| C4 | 无残影 | 增量擦除绘制应干净更新，无旧像素残留 |

### 变化测试

| 操作 | 预期效果 |
|------|---------|
| 改频率为 100 Hz (10ms 周期) | 斜坡只覆盖半屏 (读出 5ms = 半周期) |
| 改频率为 400 Hz (2.5ms 周期) | 出现两段斜坡 |
| 改为正弦波 | LCD 显示正弦曲线 |
| 改为方波 (1kHz) | LCD 显示多段阶梯 |

## 测试 D：双通道切换验证

**目的**：确认 S2 切换通道后，ADC 正确读取另一通道。

### 接线

- W1 → PB18 (J3 AO, CCD1)，设为 DC 1.0V
- W2 → PB17 (J17 AO, CCD2)，设为 DC 2.5V

### 操作步骤

1. 进入 CCD Test，默认 CH1
2. 观察 LCD：Avg ≈ 1241 (1.0V)，波形在低位
3. 按 S2 切换到 CH2
4. 观察 LCD：Avg ≈ 3103 (2.5V)，波形跳到高位
5. 再按 S2 切回 CH1
6. 观察 LCD：Avg 回到 ≈ 1241

### 判定标准

- 切换后波形**立即**反映另一通道的值（不是渐变）
- 切换时 `ccd_prev` 重置触发全帧重绘（无残影）
- CH1/CH2 标签在 LCD 正确更新

## 测试 E：曝光时间与帧率

**目的**：确认曝光调节有效、帧率合理。

### 方法

用 DIO 0 监测 SI 信号，测量**两次 SI 上升沿的间隔**即为帧周期。

### 预期帧周期

```
帧周期 ≈ flush(~1ms) + 曝光(ms) + 读出(~5ms) + LCD绘制 + loop delay(10ms)
```

| 曝光 | 预期帧周期 |
|------|-----------|
| 1 ms | ~17 ms (≈59 fps) |
| 10 ms | ~26 ms (≈38 fps) |
| 50 ms | ~66 ms (≈15 fps) |
| 100 ms | ~116 ms (≈9 fps) |

### 操作

1. 连续模式下，用 S0/S1 调整曝光
2. 在 Logic 中测量 SI 脉冲间隔
3. 确认间隔随曝光值线性增长

## 测试 F：单拍模式

**目的**：确认单拍模式下仅按键触发采集。

### 操作

1. 旋转编码器切换到 SNGL 模式（LCD 右下显示 "SNGL" 黄色）
2. **不按任何键** → SI/CLK 应无信号（Logic 中无触发）
3. 按 S0 → 应看到一次完整的 flush+读出序列
4. 波形更新一次后静止
5. 按 S1 → 同上
6. 按 S2 → 同上 + 通道切换

### 判定标准

- SNGL 模式下，SI/CLK 仅在按键时产生
- CONT 模式下，SI/CLK 持续周期性产生
- PUSH 任何模式下均退出（不触发采集）
